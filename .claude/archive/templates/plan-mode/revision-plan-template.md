# Revision Plan Template

1. Current State  
- Describe what currently exists.  
- Note what works well and what should be preserved.  
- Identify weak spots, pain points, or unclear areas.  

2. Drivers for Revision  
- Why is this revision needed now?  
- Is the revision triggered by feedback, new requirements, errors, or quality standards?  
- Clarify urgency vs importance.  

3. Revision Goals  
- Define the purpose of the revision.  
- What should the revised version accomplish that the current one does not?  
- Set scope boundaries (what is *in* vs *out* of revision).  

4. Key Focus Areas  
- Which dimensions need attention?  
  - Accuracy (facts, data, alignment)  
  - Structure (organization, readability)  
  - Style & Tone (consistency, accessibility)  
  - Functionality (performance, maintainability, usability)

6. Resources
- List the tools needed
- Specify references, documentation, or style guides.  

7. Subagents Needed
- List the subagents needed
- Specify the role/tasks delegated to the subagents

8. Success Markers  
- Define how success will be measured.  
- Examples: user feedback, test results, clarity improvements, compliance achieved.  

9. Risks & Trade-offs  
- Anticipate potential downsides of making changes.  
- Examples: breaking existing functionality, losing valuable content, scope creep.  

10. Safeguards & Guardrails  
- Define boundaries for the revision process.  
- How will you keep the scope contained and quality assured?  
- Examples: working in branches/drafts, changelog tracking, peer reviews.
